## 🔧 使用 ida-pro-mcp 工具的分析工作流

### 工作流 1: 快速扫描

```python
# 在 IDA Python 中执行 (py_eval 工具)
import idautils
import idaapi
import idc

# 统计函数信息
funcs = list(idautils.Functions())
print(f"Total functions: {len(funcs)}")

# 分析函数大小分布
sizes = []
named_count = 0
for func_ea in funcs:
    name = idc.get_func_name(func_ea)
    if not name.startswith("sub_"):
        named_count += 1
    func = idaapi.get_func(func_ea)
    sizes.append(func.end_ea - func.start_ea)

print(f"Named functions: {named_count}")
print(f"Avg size: {sum(sizes)/len(sizes):.0f}")
print(f"Max size: {max(sizes)}")
print(f"Min size: {min(sizes)}")
```

### 工作流 2: 控制流分析

```python
# 查找间接跳转
import idautils
import idaapi

indirect_jumps = []
for func_ea in idautils.Functions():
    func = idaapi.get_func(func_ea)
    for block in idaapi.FlowChart(func):
        for insn_ea in idautils.Heads(block.start_ea, block.end_ea):
            mnemonic = idaapi.print_insn_mnem(insn_ea)
            if mnemonic in ["br", "blr"]:
                opnd = idaapi.print_operand(insn_ea, 0)
                if opnd.startswith("x") and not opnd.startswith("x30"):
                    indirect_jumps.append((hex(insn_ea), mnemonic, opnd))

print(f"Found {len(indirect_jumps)} indirect jumps:")
for addr, mn, op in indirect_jumps[:20]:
    print(f"  {addr}: {mn} {op}")
```

### 工作流 3: 字符串交叉引用

```python
# 查找引用特定字符串的函数
import idautils
import idc

target_strings = ["AES", "encrypt", "decrypt"]

for s in idautils.Strings():
    str_val = str(s)
    if any(t in str_val.lower() for t in target_strings):
        print(f"String at {hex(s.ea)}: {str_val}")
        for xref in idautils.XrefsTo(s.ea):
            func_ea = idc.get_func_attr(xref.frm, idc.FUNCATTR_START)
            if func_ea != idc.BADADDR:
                func_name = idc.get_func_name(func_ea)
                print(f"  Referenced from: {func_name} ({hex(func_ea)})")
```

## 📊 OLLVM 混淆模式识别

### 1. 间接跳转 (Indirect Jump)

**特征：**
- 使用 `br xN` 或 `blr xN` 跳转
- 跳转地址计算复杂
- 通常伴随查表操作

**检测方法：**
```bash
# 使用 find_bytes 搜索
curl -X POST http://127.0.0.1:13337/mcp -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
        "name": "find_bytes",
        "arguments": {
            "patterns": ["D6 1F 00 C0", "D6 3F 00 C0"],
            "limit": 50
        }
    },
    "id": 5
}'
```

### 2. 控制流平坦化 (Control Flow Flattening)

**特征：**
- 一个主分发器 (dispatcher)
- 大量基本块
- 使用状态变量控制流程
- 函数体积异常大

**检测方法：**
- 查找大于 0x1000 字节的函数
- 分析基本块数量和连接关系
- 查找状态变量（频繁读写的全局或局部变量）

### 3. 花指令 (Bogus Control Flow)

**特征：**
- 永远不会执行的死代码
- 使用不透明谓词 (opaque predicate)
- 垃圾指令填充

**检测方法：**
- 查找不可达基本块
- 分析条件跳转的条件是否恒真或恒假

### 4. 指令替换 (Instruction Substitution)

**特征：**
- 简单指令被替换为复杂等价序列
- 例如: `mov x0, #1` → 复杂计算序列

## 🎯 反混淆策略

### 策略 1: 符号执行 + 动态分析

结合 unidbg 动态执行：
1. 使用 IDA 静态分析定位候选函数
2. 使用 unidbg 动态执行跟踪真实控制流
3. 对比静态和动态结果

### 策略 2: 模式匹配

基于已知的 AES 算法特征：
- S-Box 查找表
- 轮常量
- 特定的指令序列

### 策略 3: 逐层剥离

1. 先去除花指令
2. 再解决间接跳转
3. 最后恢复控制流平坦化
